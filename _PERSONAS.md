## Зачем персоны

Каждое решение проверяется вопросом: **"Кому из 5 персон это помогает?"**

- 0 персон → не делай
- 1-2 персоны → обсуди
- 3+ персон → приоритет

## Персона 1: Миша — Vibecoder

### Профиль

| Параметр | Значение |
|----------|----------|
| **Возраст** | 19-28 лет |
| **Роль** | Vibecoder / начинающий / студент |
| **Опыт кода** | 0-1 год. Может не знать что такое git hook |
| **Стек** | Python (скрипты, боты), Node.js (SaaS), React (лендинги) |
| **Claude Code** | Установил недавно, использует как "AI-помощник для всего" |
| **Проекты** | Pet-проекты, стартап-MVP, курсовые, Telegram-боты |
| **Мотивация** | "Чтобы работало". Не углубляется в инфраструктуру |
| **Откуда узнал о Doctor** | Telegram-канал, Twitter, "а что это за /doctor?" |

### Типичный день

Миша пишет Telegram-бота на Python. Claude Code генерирует код. Миша копирует API-ключ прямо в код, делает `git add .`, `git push`. Не думает про `.gitignore` — не знает зачем. Не знает что такое pre-commit hook. Никогда не запускал линтер вручную. У него нет CLAUDE.md — даже не знает что это.

### Боли

| # | Боль | Глубина | Что делает сейчас |
|---|------|---------|-------------------|
| 1 | **Слепые зоны безопасности** | Не знает что `.env` утечёт | Хранит ключи в коде |
| 2 | **Страх терминала** | "11 checks failed" → паника | Закрывает терминал |
| 3 | **Не знает терминологию** | "hook", "MCP", "SAST" — ? | Гуглит, теряет время |
| 4 | **Нет workflow** | Коммитит в main, нет тестов | Не знает что есть другой путь |

### Триггеры фрустрации

- Видит `red` КРИТИЧНО → паникует, думает проект сломан
- 42 проверки разом → "это слишком много, забей"
- Непонятное слово без объяснения → чувствует себя тупым
- Автофикс без объяснения → не понимает что произошло

### Триггеры восторга

- "Вот 3 важные вещи, остальное ОК" → "О, можно справиться!"
- Объяснение простым языком → "Теперь понимаю!"
- "Это займёт ~2 минуты" → "Легко!"
- Автофикс + объяснение → "Круто, оно само всё сделало и объяснило"

### Как Doctor должен говорить с Мишей

```
Нашёл 3 важных вещи для твоего проекта:

1. chmod 600 .env
   Сейчас .env может прочитать любая программа на компьютере.
   Эта команда ограничит доступ только для тебя.

2. Добавить .env в .gitignore
   Без этого `git add .` отправит пароли на GitHub.

3. Создать .env.example
   Чтобы ты (или кто-то другой) знал какие переменные нужны,
   но без реальных паролей.

Исправить все три? ~2 минуты.
```

| **Проекты** | Рабочие + pet, начинает работать в команде |

| **Откуда узнал** | GitHub Explore, коллега порекомендовал, Reddit |

### Типичный день

Андрей работает над FastAPI сервисом. Claude Code генерирует код, но CLAUDE.md не создавал. Тесты пишет, но coverage не измеряет. Линтер есть (ruff), но не в pre-commit. Периодически забывает запустить тесты перед push. Коммитит в main — feature branches "надо бы, но как-нибудь потом".

### Боли

| # | Боль | Глубина | Что делает сейчас |
|---|------|---------|-------------------|
| 1 | **Знает "надо" но не "как"** | Слышал про hooks, не настраивал | Надеется что "как-нибудь" |
| 2 | **Нет системы** | Тесты есть, CI нет. Lint есть, pre-commit нет | Частичная автоматизация |
| 3 | **Хочет учиться** | Каждое "почему" = обучение | Читает, но мало практикует |
| 4 | **Боится сломать** | В команде, не хочет уронить main | Перепроверяет вручную |

### Триггеры фрустрации

- "Сделай X" без объяснения почему → копирует без понимания
- Слишком много находок без приоритизации → не знает с чего начать
- Автофикс сломал что-то → потерял доверие
- Нет оценки усилий → "это на 5 минут или на 3 часа?"

### Триггеры восторга

- "Зачем: ..." объяснение при каждом фиксе → учится
- "Усилие: ~5 минут" → может планировать
- 73% → "Вот что сделать для 80%" → чёткий путь роста
- Показывает diff изменений → понимает что произошло

### Как Doctor должен говорить с Андреем

```
Безопасность: 8/11 (73%) — ХОРОШО

Что работает:
  Секреты не в git
  .gitignore покрывает стек
  .env.example синхронизирован

Что улучшить:
  Pre-commit hook — ruff check + gitleaks на staged файлы
    Зачем: можно закоммитить syntax error или секрет.
    Сейчас lint вручную — легко забыть.
    Усилие: ~5 минут

  PostToolUse hook — auto-lint при каждом Edit/Write Claude
    Зачем: Claude иногда генерирует баги.
    PostToolUse ловит сразу, не при коммите.
    Усилие: ~3 минуты

Исправить оба? Или по одному?
```

## Как использовать этот документ

### При добавлении фичи
1. Посчитай Score в матрице
2. Score = 0 → не добавляй
3. Score >= 10 → приоритет
4. Проверь: помогает 3+ персонам?

### При изменении формата вывода
1. Прочитай "Как Doctor должен говорить с [персона]"
2. Миша → подробно + мягко
3. Дима → кратко + сухо
4. Текущий формат = баланс для Андрея/Кати

### При выборе severity
1. Миша запаникует от лишнего `red`?
2. Дима проигнорирует лишний `blue`?
3. Используй правило из _VISION.md: "Что случится если НЕ исправить?"

### При написании объяснений
1. Миша: "Эта команда ограничит доступ только для тебя"
2. Андрей: "Зачем: без этого можно закоммитить syntax error"
3. Катя: "coverage threshold >=80%"
4. Дима: (без объяснений)
5. Лена: "Новый contributor не сможет начать"

### При приоритизации
1. Суммарный балл из матрицы
2. Формула из _VISION.md: `Impact × Reach × Frequency / Effort`
3. Если конфликт → Score из матрицы побеждает (ближе к реальным юзерам)
